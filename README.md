# Noqlen Flux Core

Noqlen Flux Core is the future core for search, download, staging, validation, and handoff workflows in the Noqlen ecosystem.

This repository is in its initial bootstrap phase. It does not perform real downloads, imports, network calls, library writes, cleanup, or automatic deletion.

## Lineage

- Noqlen Forge Core is the architectural reference for service-first design, safety, docs, tests, and release hardening.
- The legacy `slsk` project is a reference and laboratory for search/download/staging ideas, not a direct base for Flux.
- Flux is a separate project and should not be treated as a continuation of `slsk`.

## Current Scope

- A minimal Python package named `noqlen_flux`.
- A public future CLI entry point named `noqlen-flux`.
- Initial structured result contracts and a service-first base.
- Safe workspace inspection and initialization primitives.
- Safe JSON and text report artifact generation from structured service results.

No operation currently performs real search, downloads, network calls, imports, cleanup, or music library writes.

## Safe Workspace

Flux-owned staging directories can be planned without writing anything:

```bash
noqlen-flux workspace init ./flux-workspace --dry-run
```

Apply mode must be explicit before directories are created:

```bash
noqlen-flux workspace init ./flux-workspace --apply
```

Workspace initialization creates only Flux-controlled directories such as `incoming`, `approved`, `quarantine`, `rejected`, `reports`, `manifests`, `cache`, and `tmp`, after path containment and symlink safety checks.

## Safe Reports

Reports are generated by core services from `FluxResult` data. The CLI only requests preview or writing and renders the returned result.

Preview a safe demo report without writing a file:

```bash
noqlen-flux report demo --workspace ./flux-workspace --format json --dry-run
```

Apply mode must be explicit before a report is written:

```bash
noqlen-flux report demo --workspace ./flux-workspace --format json --apply
```

Report files are confined to `workspace/reports` and do not perform network calls, downloads, imports, cleanup, or music library writes.
